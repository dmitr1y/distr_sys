 Потокобезопасная хеш-таблица с закрытой адресацией.
 
 Потокобезопасная структура данных должна поддерживать
 операции добавления, удаления и поиска элементов. Необходимо
 разработать тестовую программу для проверки работоспособно-
 сти структуры данных. Алгоритм работы тестовой программы:
 1. Создаётся p тестовых параллельных потоков.
 2. Каждый поток выполняет n операций (вид операций выбира-
 ется случайным образом) для случайных элементов.
 3. Измеряется время выполнений операций. Для корректного
 измерения времени необходимо синхронизировать потоки
 перед началом тестовых операций (см. приложение 6).
